# GUIでGitを使おう

## はじめに

### CLIとGUIの使い分け

これまでのレッスンでは、ターミナル（CLI）を使ったGit操作を学んできました。CLIはGitのすべての機能にアクセスでき、操作の自動化にも向いています。

一方、**GUI（グラフィカルユーザーインターフェース）** を使えば、マウス操作で直感的にGitを使うことができます。Git対応のGUIツールにはさまざまなものがありますが、本記事では最も広く使われているコードエディタ **VS Code（Visual Studio Code）** を使って解説します。

| 観点 | CLI（ターミナル） | GUI（VS Code） |
|------|-------------------|-----------------|
| 操作感 | コマンド入力 | マウス＋ボタン |
| 差分確認 | `git diff`（テキスト） | 色付きの並列ビュー |
| コンフリクト解決 | テキストエディタで手動編集 | ボタンで選択（Accept Current/Incoming） |
| 高度な操作 | すべてのGitコマンドが使える | 基本操作が中心 |
| 学習効果 | Gitの仕組みを深く理解できる | 視覚的にわかりやすい |

**おすすめの使い分け**：日常の基本操作（ステージング、コミット、プッシュ）はVS Codeで行い、複雑な操作（リベース、チェリーピックなど）はCLIで行うのが効率的です。

## 準備

### VS Codeのインストール

VS Codeをまだインストールしていない場合は、以下の手順でセットアップしましょう。

1. [VS Code公式サイト](https://code.visualstudio.com/) にアクセス
2. お使いのOS（Windows / macOS / Linux）に合ったインストーラーをダウンロード
3. ダウンロードしたファイルを実行し、画面の指示に従ってインストール
4. インストール完了後、VS Codeを起動

> **Tips**：VS CodeにはGit機能が標準で組み込まれているため、追加のインストールは不要です。ただし、**Git本体**がPCにインストールされている必要があります。ターミナルで `git --version` を実行して、Gitが使えることを確認しておきましょう。

### Source Controlパネルを開く

VS Codeの左側サイドバーにある**分岐アイコン**（Source Control）をクリックするか、ショートカットキーで開きます。

| OS | ショートカット |
|----|----------------|
| macOS | `Ctrl + Shift + G` |
| Windows | `Ctrl + Shift + G` |

### パネルの見方

Source Controlパネルには以下の情報が表示されます。

![Source Controlパネル全体の外観](/images/git-vscode/source-control-panel.png)

- **M**（Modified）：変更されたファイル
- **U**（Untracked）：新規作成されたファイル
- **D**（Deleted）：削除されたファイル
- **A**（Added）：ステージに追加済み

## 基本操作

### ファイルの変更確認（差分ビュー）

ファイルを編集して保存すると、Source Controlパネルの「Changes」に表示されます。

**ファイル名をクリック**すると、差分ビュー（diff view）が開きます。

![差分ビュー（サイドバイサイド表示）](/images/git-vscode/diff-view.png)

- 赤色（`-`）：削除された行
- 緑色（`+`）：追加された行

> **Tips**：差分ビュー右上のアイコンで「インライン表示」と「サイドバイサイド表示」を切り替えられます。

### ステージング（+ボタン）

コミットしたいファイルをステージングエリアに追加します。

- **個別にステージング**：ファイル名の右にある **`+`** ボタンをクリック
- **すべてステージング**：「Changes」の右にある **`+`** ボタンをクリック
- **ステージング解除**：「Staged Changes」のファイル右にある **`-`** ボタンをクリック

![+ボタンでファイルをステージングする操作](/images/git-vscode/staging.png)

### コミット（メッセージ入力 → チェックボタン）

1. Source Controlパネル上部の**メッセージ入力欄**にコミットメッセージを入力
2. **✓（チェック）ボタン**をクリック、または `Cmd + Enter`（macOS）/ `Ctrl + Enter`（Windows）

![メッセージ入力欄とコミットボタン](/images/git-vscode/commit.png)

> **注意**：ステージングしていないファイルがある場合、「ステージされた変更がありません。すべての変更をステージしてコミットしますか？」と確認ダイアログが表示されます。

### 変更の取り消し

間違った変更を元に戻したい場合：

- **ファイル単位で取り消し**：ファイル右の **↩（Discard Changes）** ボタンをクリック
- **すべての変更を取り消し**：「Changes」右の **↩** ボタンをクリック

![Discard Changesボタンの位置](/images/git-vscode/discard-changes.png)

> **注意**：取り消した変更は復元できません。重要な変更がある場合は、先にコミットするかスタッシュしてください。

### コミット履歴の確認（タイムラインビュー）

VS Codeでコミット履歴を確認する方法はいくつかあります。

**タイムラインビュー**：

1. エクスプローラー（左サイドバー）でファイルを選択
2. 下部に表示される「TIMELINE」パネルを確認
3. 各コミットをクリックすると、そのコミットでの変更差分が表示される

![タイムラインビューでのコミット履歴表示](/images/git-vscode/timeline.png)

## ブランチ操作

### ブランチの作成・切り替え（ステータスバー）

VS Code画面**左下のステータスバー**に、現在のブランチ名が表示されています。

ブランチ名をクリックすると、以下の操作メニューが表示されます。

- **既存ブランチへの切り替え**：一覧からブランチを選択
- **新しいブランチの作成**：「Create new branch...」を選択し、ブランチ名を入力
- **新しいブランチを作成して切り替え**：「Create new branch from...」を選択

![ステータスバーのブランチ名クリック → ブランチ選択メニュー](/images/git-vscode/branch-switch.png)

### ブランチの一覧と管理

コマンドパレット（`Cmd + Shift + P` / `Ctrl + Shift + P`）から「Git: Branch」と入力すると、ブランチ関連のコマンドが一覧表示されます。

- **Git: Create Branch** - 新しいブランチを作成
- **Git: Delete Branch** - ブランチを削除
- **Git: Rename Branch** - ブランチ名を変更
- **Git: Checkout to** - ブランチを切り替え

---

## マージとコンフリクト解決

### ブランチのマージ

1. マージ先のブランチに切り替える（例：`main`）
2. コマンドパレット（`Cmd + Shift + P` / `Ctrl + Shift + P`）を開く
3. 「**Git: Merge Branch...**」を選択
4. マージするブランチを選択（例：`feature/login`）

![コマンドパレットでGit: Merge Branch...を選択](/images/git-vscode/merge-branch.png)

### コンフリクトの視覚的解決

マージ時にコンフリクトが発生すると、VS Codeは対象ファイルを**特別なビュー**で表示します。

![コンフリクトビュー（Accept Current/Incoming/Bothボタン表示）](/images/git-vscode/conflict-resolve.png)

コンフリクト箇所の上部に、以下の**操作ボタン**が表示されます。

| ボタン | 動作 |
|--------|------|
| **Accept Current Change** | 現在のブランチ（HEAD）の内容を採用 |
| **Accept Incoming Change** | マージ元ブランチの内容を採用 |
| **Accept Both Changes** | 両方の内容を残す |
| **Compare Changes** | 差分を並べて比較 |

> **Tips**：ボタンで選択した後、必要に応じて手動で微調整することもできます。解決が完了したら、ファイルを保存してステージング → コミットを行います。

## リモート操作

### プッシュ・プル・同期

VS Codeでは、リモートリポジトリとの同期を簡単に行えます。

**ステータスバーの同期ボタン**：

![ステータスバーの同期ボタン（↑↓表示）](/images/git-vscode/sync-button.png)

- **↑（上矢印）の数字**：ローカルにあるがリモートにプッシュされていないコミット数
- **↓（下矢印）の数字**：リモートにあるがローカルにプルされていないコミット数
- **🔄（同期ボタン）**：プルとプッシュをまとめて実行

**個別の操作**：

Source Controlパネルの **「...」メニュー** から以下を選択できます。

- **Push** - ローカルの変更をリモートにプッシュ
- **Pull** - リモートの変更をローカルにプル
- **Fetch** - リモートの情報を取得（マージはしない）
- **Push to...** - プッシュ先のリモートを指定

### クローンとリモート追加

**リポジトリのクローン**：

1. コマンドパレット → 「**Git: Clone**」を選択
2. リポジトリのURLを入力
3. クローン先のフォルダを選択

![コマンドパレットでGit: Cloneを選択](/images/git-vscode/clone.png)

## おすすめ拡張機能

VS Codeの標準Git機能をさらに強化する拡張機能を紹介します。

### GitLens

**最も人気のあるGit拡張機能**で、以下の機能が追加されます。

- **行ごとのblame表示**：各行の最終変更者・日時・コミットメッセージをインラインで表示
- **コミット履歴の可視化**：ファイルやブランチの詳細な履歴をツリー表示
- **比較機能**：ブランチ間・コミット間の差分を簡単に確認

```text
インストール方法：
  1. 拡張機能パネル（Cmd/Ctrl + Shift + X）を開く
  2. 「GitLens」で検索
  3. インストールをクリック
```

### Git Graph

**ブランチの視覚的なグラフ表示**ができる拡張機能です。

- ブランチの分岐・マージを視覚的に確認
- コミット間の関係を一目で把握
- グラフ上からチェックアウトやマージなどの操作が可能

```text
使い方：
  1. コマンドパレット → 「Git Graph: View Git Graph」を選択
  2. または、Source Controlパネルのタイトルバーにあるグラフアイコンをクリック
```

## CLIとの使い分けガイド

実務では、操作の種類に応じてCLIとGUIを使い分けるのが効率的です。

### VS Code（GUI）が便利な場面

| 操作 | 理由 |
|------|------|
| 差分の確認 | 色付きの並列ビューでわかりやすい |
| ステージング＋コミット | マウス操作で素早くできる |
| コンフリクト解決 | ボタン選択で直感的 |
| ブランチの切り替え | ステータスバーからワンクリック |
| 変更の取り消し | 対象ファイルが視覚的に確認できて安全 |

### CLI（ターミナル）が便利な場面

| 操作 | 理由 |
|------|------|
| リベース | GUIでは操作が複雑になりがち |
| チェリーピック | 特定コミットの操作はCLIが正確 |
| リフログ | 詳細な履歴調査はCLIが得意 |
| 複雑なリセット | `--soft`/`--mixed`/`--hard` の指定はCLI向き |
| スクリプト・自動化 | CIや自動化にはCLIが必須 |
| Git設定の変更 | `.gitconfig` の編集はCLIが効率的 |

### VS Code内でターミナルを使う

VS Codeには**統合ターミナル**が内蔵されています。GUIとCLIを同じ画面で使い分けられます。

| OS | ショートカット |
|----|----------------|
| macOS | `` Ctrl + ` `` |
| Windows | `` Ctrl + ` `` |

> **おすすめの使い方**：普段の操作はSource Controlパネルで行い、複雑な操作が必要なときだけ統合ターミナルに切り替える、というワークフローが効率的です。

## まとめ

- VS Codeには**Git機能が標準搭載**されており、基本操作はマウスだけで行える
- **差分確認**と**コンフリクト解決**はGUIが特に便利
- **ブランチ操作**はステータスバーとコマンドパレットから簡単にアクセスできる
- **GitLens**や**Git Graph**などの拡張機能でさらに機能を強化できる
- CLIとGUIは**それぞれの得意分野**で使い分けるのが実務では効率的
- VS Codeの**統合ターミナル**を活用すれば、GUIとCLIをシームレスに切り替えられる
