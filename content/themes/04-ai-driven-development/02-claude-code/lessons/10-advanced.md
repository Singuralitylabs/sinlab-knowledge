# 高度な機能

権限制御、思考モード、セッション移動など、Claude Code をもう一歩踏み込んで使いこなすための機能を紹介します。日常のコーディングから、設計判断や長時間タスクの運用まで、場面に応じた切り替え方を身につけましょう。

## このページで学べること

- 4 種類の権限モード（default / acceptEdits / plan / bypassPermissions）の違いと使い分け
- Extended Thinking と `think` キーワードによる深い分析の呼び出し方
- `/model` コマンドと Effort レベルによるコストと精度の調整
- Bash モード（`!` プレフィックス）と `@` ファイルメンションの使い方
- Vim モード、プロンプトサジェスト、チェックポイント（Esc Esc）などの入力支援
- セッション移動（`--resume` / `-c` / `/teleport` / `/desktop` / `--remote`）で作業を引き継ぐ方法

## 権限モード（4 種類）

Claude Code は操作の承認レベルを 4 種類から選べます。作業内容やリスクに応じて切り替えることで、安全性と効率性を両立させます。プロンプト入力中に `Shift+Tab` を押すと、モードを順番に切り替えられます。

| モード名 | 内部名 | ファイル変更 | コマンド実行 | 主なシーン |
| --- | --- | --- | --- | --- |
| Default | `default` | 都度確認 | 都度確認 | 通常の開発作業 |
| Accept Edits | `acceptEdits` | 自動承認 | 都度確認 | 信頼できる編集作業 |
| Plan | `plan` | 不可 | 不可 | 事前に計画を確認したい場面 |
| Bypass Permissions | `bypassPermissions` | 自動承認 | 自動承認 | 検証済みの自動化スクリプト |

> [!WARNING]
> `bypassPermissions` はすべての確認をスキップします。信頼できるサンドボックスや検証済みの自動化でのみ使用し、未知のリポジトリや本番環境では避けてください。

### Shift+Tab でモードを切り替え

プロンプト入力中に `Shift+Tab` を押すと、権限モードが順番に切り替わります。現在のモードはプロンプトの左側にインジケーターとして表示されます。

```text
Default → Accept Edits → Plan → Bypass → Default …
```

### おすすめワークフロー

```text
Step 1: Plan モードで計画を確認
   ↓
Step 2: Default / Accept Edits で実行
   ↓
Step 3: 結果を確認して次のタスクへ
```

> [!TIP]
> 大きな変更の前は Plan モードで計画を確認してから実行するのがおすすめです。Claude がファイルを書き換えずに提案だけを返すため、意図しない変更を未然に防げます。

## Extended Thinking（拡張思考）

複雑な問題に取り組む際、Claude により深い分析を行わせる拡張思考モードです。アーキテクチャの設計判断、複雑なデバッグ、マルチファイルのリファクタリングなど、通常の応答よりも深い推論が必要な場面で効果を発揮します。

### Alt+T でトグル、または think キーワード

プロンプト入力中に `Alt+T` を押すと、拡張思考モードのオン / オフを切り替えられます。有効にすると、Claude は回答の前に内部で長い思考プロセスを実行し、より精度の高い回答を返します。

また、プロンプト内に `think` / `think hard` / `think harder` / `ultrathink` といったキーワードを含めることで、思考の深さを段階的に指示することもできます。

```text
このアーキテクチャの選択肢を think hard で比較してください
```

### 使うと効果的な場面

- アーキテクチャの設計判断 — 複数の選択肢からベストな設計を選ぶとき
- 複雑なデバッグ — 原因が特定しにくいバグの調査
- マルチファイルのリファクタリング — 広範囲にわたるコード変更の計画
- セキュリティレビュー — 脆弱性の網羅的な分析

## Effort レベル調整

一部のモデル（例: Opus 4.x 系）では、Claude の思考の深さ（Effort レベル）を調整できます。簡単な質問には低い Effort で素早く応答し、複雑な問題には高い Effort で深い分析を行うことで、コストと精度のバランスを最適化します。

### 操作方法

`/model` コマンドを実行後、矢印キー（← →）で Effort レベルを調整します。

```text
/model
  ↑↓: モデル選択
  ←→: Effort レベル調整（低 ↔ 高）
```

| レベル | コスト | 応答速度 | 向く用途 |
| --- | --- | --- | --- |
| 低 Effort | 低 | 速い | 軽い質問、小さな修正 |
| 中 Effort | 標準 | 標準 | 通常の開発作業 |
| 高 Effort | 高 | 遅い | 複雑な設計、大規模リファクタリング |

## Bash モード（`!` プレフィックス）

入力の先頭に `!` を付けると、シェルコマンドを直接実行できます。Claude の解釈や承認プロセスを経由せず、即座にコマンドが実行され、その結果が会話コンテキストに追加されます。

```bash
! npm test
! git status
! ls -la src/
```

| 観点 | `!` 直接実行 | 通常の Bash ツール（Claude 経由） |
| --- | --- | --- |
| 実行プロセス | 即座に実行 | Claude が判断して実行 |
| 承認 | 不要 | 権限モードに依存 |
| Claude の解釈 | なし | あり（コマンドを選択・構築） |
| 結果の活用 | コンテキストに追加 | コンテキストに追加 |
| 向く用途 | 既知のコマンドを素早く実行 | 適切なコマンドを Claude に任せたい場合 |

> [!TIP]
> `!` で実行した結果は会話に含まれるため、続けて「このテスト結果を分析して」のように Claude に指示を出すことができます。

## `@` ファイルメンション

会話中に `@` を入力すると、ファイルパスのオートコンプリートが起動します。特定のファイルを素早く参照し、Claude にそのファイルの内容を認識させた上で指示を出せます。

```text
@src/auth.ts のログイン関数を修正して

@package.json の依存関係を確認して

@README.md を最新の変更に合わせて更新して
```

### ファイルメンションのメリット

- オートコンプリートにより、長いパスを手入力する必要がない
- Claude がファイルの内容を正確に把握した上で作業を行える
- 複数のファイルを同時にメンションして、関連ファイルをまとめて参照可能

## Vim モード

`vim` キーバインドに慣れた人向けに、Vim モードが提供されています。設定を有効化すると、プロンプト入力で `ESC` によるノーマルモードへの切り替えや、`hjkl` での移動、`dd` / `cw` などのオペレーターが使えるようになります。

```bash
# 設定コマンドから有効化
/vim
```

> [!NOTE]
> Vim モードはあくまでプロンプトの入力支援で、Claude のエージェント動作そのものは変化しません。自分の入力速度を上げたい人向けの機能です。

## プロンプトサジェスト

Claude Code は Git の履歴や会話の文脈を分析して、次に実行すべきアクションを自動的に提案します。プロンプトキャッシュを再利用するため、追加コストはわずかです。

```text
Git 履歴 + 会話文脈 → AI 分析 → 次のアクション提案 → Tab で採用
```

| キー | 動作 |
| --- | --- |
| `Tab` | サジェストを採用（プロンプトに入力） |
| `Enter` | サジェストを即座に実行 |

> [!TIP]
> サジェストはプロンプトキャッシュを再利用するため、追加のトークンコストはごくわずかです。積極的に活用して作業効率を向上させましょう。

## チェックポイント（Esc Esc）

`Esc` を 2 回連続で押すと、直前のチェックポイントまで巻き戻せます。ファイル編集の途中で「やっぱり 1 ステップ前に戻したい」というときに有効です。連続で押すことで、より古いチェックポイントに遡れます。

```text
Esc Esc         … 1 つ前のチェックポイントへ
Esc Esc Esc Esc … さらに前のチェックポイントへ
```

> [!IMPORTANT]
> チェックポイントはファイル変更の論理的な切れ目で作られますが、Git のコミットの代わりではありません。重要な区切りでは通常通りコミットしておくのが安全です。

## セッション間移動と継続

Claude Code は、過去のセッションを呼び出したり、別の実行環境（CLI / Web / Desktop）で作業を続けたりできます。作業環境を切り替えながら、会話の文脈を途切れさせずに進められます。

### ローカル CLI でのセッション再開

```bash
# 直前のセッションを継続
claude -c

# 過去のセッションを選んで再開
claude --resume
```

- `claude -c`（`--continue`）: 直前のセッションを即座に再開します。
- `claude --resume`: インタラクティブにセッションを選んで再開します。

### CLI / Web / Desktop をまたいだ移動

| コマンド | 方向 | ユースケース |
| --- | --- | --- |
| `/teleport` | Web ↔ CLI | Web（claude.ai）で始めた作業をローカル CLI に移行。ローカルのファイルシステムへ直接アクセスしたい場合に便利 |
| `/desktop` | CLI ↔ Desktop | CLI での作業を Desktop アプリに移行。ビジュアルな diff レビューや複数セッションの並行確認に便利 |
| `--remote` | CLI → Web | ローカルで開始した長時間タスクを Web に移行。ターミナルを閉じても処理が継続するため、ビルドやテストの長時間実行に最適 |

> [!TIP]
> 短時間のスクラッチ作業は CLI、レビューや会議中は Desktop、出先での確認は Web と、シーンに応じて切り替えると生産性が上がります。

## まとめ: 場面に応じた使い分け

| 目的 | おすすめの機能 |
| --- | --- |
| 事前に計画だけ確認したい | Plan モード |
| 単調な編集を素早く回したい | Accept Edits モード |
| 深い分析をさせたい | Extended Thinking / `think` キーワード |
| コストを下げたい | Effort レベルを下げる |
| 既知のコマンドをすぐ試したい | `!` Bash モード |
| 特定ファイルを参照させたい | `@` ファイルメンション |
| 誤編集を戻したい | `Esc Esc`（チェックポイント） |
| 前の会話を続けたい | `claude -c` / `--resume` |
| 環境を変えて続けたい | `/teleport` / `/desktop` / `--remote` |
