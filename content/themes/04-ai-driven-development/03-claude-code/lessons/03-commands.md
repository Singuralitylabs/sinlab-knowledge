# コマンド & ショートカット

Claude Code の対話モードや CLI から呼び出せるコマンドを、カテゴリ別にまとめたチートシートです。スラッシュコマンド、CLI フラグ、キーボードショートカット、権限モードの 4 つに分けて解説します。

## なぜコマンドを覚えると良いのか

Claude Code は自然言語で指示するだけでも十分に使えますが、コマンドを活用することで**作業効率が大きく向上**します。たとえば、会話が長くなってきた時に `/compact` で要約すればコンテキストを節約でき、`/context` でコンテキスト使用量を確認すれば会話の管理も簡単です。コマンドは一度覚えてしまえばすぐに使えるものばかりなので、まずはよく使うものから試してみましょう。

### まずはこの 5 つを覚えよう

| コマンド | 役割 |
|---|---|
| `/init` | 最初に実行。`CLAUDE.md` を生成してプロジェクトの文脈を Claude に伝えます |
| `/context` | コンテキスト使用量を可視化。残量を把握して会話を管理 |
| `claude -c` | 直前の会話を再開。作業を中断しても文脈を引き継げる |
| `claude -r` | 過去の会話一覧から選んで再開。複数タスクの並行作業に便利 |
| `Ctrl + C` | 処理を途中で止める。意図しない動作をすぐに中断できる |

> [!TIP]
> `/help` を実行すれば、現在のバージョンで利用可能なスラッシュコマンドの最新一覧をその場で確認できます。

## スラッシュコマンド

対話中に使えるビルトインコマンドです。プロンプトの先頭に `/` を付けて入力します。

| コマンド | 説明 | 使い方 | Tip |
|---|---|---|---|
| `/help` | ヘルプ情報を表示 | `/help` | 利用可能なコマンド一覧を確認できます |
| `/init` | プロジェクト初期設定 (`CLAUDE.md` 生成) | `/init` | プロジェクトのルートで実行すると `CLAUDE.md` が作成されます |
| `/clear` | 会話履歴をクリア | `/clear` | 別の話題に切り替える時にコンテキストをリセット |
| `/compact` | 会話を要約して圧縮 | `/compact` | 長い会話でコンテキストが不足した時に便利 |
| `/exit` | Claude Code を終了 | `/exit` | セッションを明示的に終了 |
| `/model` | 使用するモデルを切り替え | `/model` | Sonnet と Opus を切り替えられます |
| `/context` | コンテキスト使用量をカラーグリッドで可視化 | `/context` | コンテキストが埋まると応答品質が低下するため定期的に確認を |
| `/cost` | トークン使用量の統計を表示 | `/cost` | サブスクリプションプランではプラン情報のみ表示されます |
| `/memory` | `CLAUDE.md` メモリを編集 | `/memory` | プロジェクト固有の指示を永続化できます |
| `/config` | 設定の表示・変更 | `/config` | `settings.json` の内容をインタラクティブに確認 |
| `/permissions` | 権限設定の表示・変更 | `/permissions` | 許可/拒否リストの追加・削除 |

> [!NOTE]
> `/compact` と `/clear` は似ていますが目的が違います。`/compact` は会話の要約を残したまま圧縮、`/clear` は履歴を完全に消去します。タスクを継続するなら `/compact`、完全に切り替えるなら `/clear` を選びましょう。

## CLI フラグ

ターミナルから `claude` を起動するときに指定するオプションです。シェルスクリプトや CI との組み合わせで威力を発揮します。

| コマンド | 説明 | 使い方 | Tip |
|---|---|---|---|
| `claude` | 対話モードを開始 | `claude` | 引数なしで実行すると対話モードに入ります |
| `claude "質問"` | 質問を指定して対話を開始 | `claude "このプロジェクトの構成を教えて"` | 初期プロンプト付きで対話モードを開始 |
| `-p / --print` | 非対話モード (結果を出力して終了) | `claude -p "この関数を説明して"` | スクリプトやパイプラインでの利用に最適 |
| `-c / --continue` | 前回の会話を再開 | `claude -c` | 直前のセッションの文脈を引き継ぎます |
| `-r / --resume` | 過去の会話を選んで再開 | `claude -r` | 会話の一覧が表示され、再開したいものを選べます |
| `--model` | 使用するモデルを指定 | `claude --model sonnet` | 起動時にモデルを固定したいとき |

```bash
# 非対話モードのパイプ例
cat error.log | claude -p "このエラーの原因を 3 行で説明して"

# 直前のセッションを引き継いで続きから作業
claude -c

# 過去の任意のセッションを選択して再開
claude -r
```

## キーボードショートカット

対話中に使える便利なキー操作です。長時間の処理の中断や複数行入力に役立ちます。

| ショートカット | 説明 | 使い方 | Tip |
|---|---|---|---|
| `Ctrl + C` | 現在の処理を中断 | 実行中に `Ctrl + C` | 長時間の処理を途中で止めたい時に |
| `Ctrl + D` | Claude Code を終了 | プロンプトで `Ctrl + D` | `/exit` と同等 |
| `Ctrl + L` | 画面をクリア | プロンプトで `Ctrl + L` | 履歴はそのまま、表示だけ綺麗にする |
| `\ + Enter` | 複数行入力モード | テキスト入力中に `\ + Enter` | 長い指示を改行しながら入力できます |
| `Option + Enter` | 複数行入力モード (Mac) | テキスト入力中に `Option + Enter` | Mac ユーザー向けの代替ショートカット |
| `Shift + Tab` | 権限モードの切り替え | プロンプトで `Shift + Tab` | Plan モードと通常モードを素早く切り替え |

> [!TIP]
> 複数行の指示を書きたい時は、バックスラッシュ + Enter (または Mac なら Option + Enter) で改行できます。仕様書レベルの長い依頼を貼り付けるときに便利です。

## 権限モード

ファイル操作やコマンド実行の許可レベルを決めるモードです。安全性と作業効率のバランスを取るために使い分けます。

| モード | 説明 | 使い方 | Tip |
|---|---|---|---|
| `default` | 操作のたびに確認を求める (デフォルト) | 初回起動時の標準設定 | 安全性を重視する場合におすすめ |
| `acceptEdits` | ファイル編集を自動承認 | `claude --allowedTools edit,write` | ファイル変更の都度確認が不要になります |
| `plan` | 計画のみ作成し、実際の変更は行わない | `Shift + Tab` で切り替え | 大きな変更前に計画を確認したい時に便利 |

> [!WARNING]
> `acceptEdits` は編集の確認をスキップするため、意図しない変更が加わるリスクが上がります。Git の作業ブランチ上で使う、範囲を限定したタスクで使うなど、運用ルールを決めてから利用しましょう。

> [!IMPORTANT]
> `plan` モードでは Claude が提案のみを行い、ファイル書き換えやコマンド実行を行いません。リファクタリングやマイグレーションなど、大きな変更の前段で「まず計画だけ立てる」用途に最適です。
